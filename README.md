# Barber App

**Barber App** é um projeto de portfólio desenvolvido para demonstrar habilidades em **Java**, **Spring Boot**, **MongoDB**, **Docker** e integração com frontend via **React**.  

O objetivo do projeto é criar um sistema funcional de gerenciamento de barbearia, permitindo o cadastro e gerenciamento de **Clientes, Usuários, Serviços** e **Agendamentos**, com CRUD completo, validações básicas e testes via **Postman**.  

Este projeto foi estruturado como uma aplicação prática para exibir competências em desenvolvimento backend e frontend, organização de código, boas práticas de programação e integração de sistemas.  

Projeto em desenvolvimento com foco no CRUD de **Clientes, Usuários e Serviços**, incluindo validações básicas e testes via **Postman**.

## Cronograma do Projeto

### Semana 1: Backend inicial (CONCLUIDO)
- Criar entidades: **Clientes, Usuários e Serviços**  
- Implementar **CRUD completo** para cada entidade  
- Adicionar **validações básicas**  
- Criar **DTOs, Services, Impl e Exceptions**  
- Testar os endpoints via **Postman**  

---

### Semana 2: Backend avançado + autenticação
- Criar CRUD de agendamentos  
- Testar conflitos de horário (regra básica)  
- Implementar login básico de administrador (JWT ou sessão simples)  
- Validar fluxos de negócios (ex.: não agendar fora do horário)  
- Testar todas as APIs com Postman e corrigir bugs  
- Preparar documentação inicial do backend (README, endpoints)  
- Revisar todo backend e realizar teste final de integração entre clientes, serviços, funcionários e agendamento  
- Entrega parcial do backend completo para MVP  

---

### Semana 3: Frontend básico + integração
- Revisar React e conceitos básicos de componentes, estado e props  
- Criar telas principais: login e dashboard inicial  
- Tela de cadastro/listagem de clientes e serviços  
- Tela de cadastro/listagem de funcionários  
- Tela de agendamentos (simples, lista de horários)  
- Integrar frontend com backend via APIs  
- Testar cada fluxo funcional (CRUD)  
- Ajustes de layout e design mínimo (limpo e funcional)  
- Entrega parcial do frontend funcional  

---

### Semana 4: Refinamento + documentação
- Revisar fluxos completos (criar, editar, deletar, agendar)  
- Corrigir bugs críticos  
- Melhorar layout e responsividade mínima  
- Documentação detalhada: README com instruções, prints e screenshots  
- Preparar testes finais do sistema (simular agendamentos e cadastros)  
- Ajustes finais em funcionalidades e design  
- Revisão completa do MVP: backend + frontend  
- Entrega do MVP funcional pronto para portfólio  

---

### Semana 5: Deploy e finalização
- Deploy do backend (Heroku ou outro)  
- Deploy do frontend (Vercel, Netlify)  
- Teste completo online do sistema  
- Ajustes pós-deploy (bugs, layout, links)  
- Preparar material para portfólio (prints, descrição do projeto)  
- Criar vídeo curto ou documentação visual do funcionamento  
- Finalização e revisão geral. Projeto pronto para mostrar
